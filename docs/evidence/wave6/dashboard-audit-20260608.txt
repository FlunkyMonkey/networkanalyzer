ROOT CAUSE 1 — .keyword mapping rot (8 flow panels, dark ~3 weeks)
------------------------------------------------------------------
Phase 3a (2026-05-03) changed the index template to map the enrichment fields
as native `keyword`. Phase 4 dashboards used `.keyword` paths, valid only while
the older dynamically-mapped (text + .keyword) indices remained in retention.
Those aged out of the 14-day ISM window ~2026-05-18. From then on, every panel
aggregating on `<field>.keyword` returned zero buckets — silently.

Proof on flows-2026.06.08:
  dst_app_label mapping: {"type": "keyword"}
  terms agg on dst_app_label.keyword -> 0 buckets
  terms agg on dst_app_label         -> 3 buckets (data)

Fix: all 46 `.keyword` references in grafana-flow-dashboards.yaml flipped to
plain field paths (bucketAggs fields, renameByName keys, byName matchers).
Design-doc caveat marked RESOLVED with the recorded lesson: a dashboard data
audit must follow any retention-driven mapping transition. A nightly automated
panel audit is proposed in the feature backlog.
